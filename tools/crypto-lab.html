Engineering & Security Notes

  • Hash Verification: The "Matcher" performs a case-insensitive, O(n) comparison against all generated outputs. A match proves integrity (data hasn't changed), not authenticity (who sent it).
  • Cryptographic Brokenness: MD5 and SHA-1 are cryptographically broken (collision attacks exist). Use them only for legacy checksums or non-security identifiers. Use SHA-256/SHA-3 for security.
  • Raw AES Keys: In "Raw" mode, keys and IVs are validated by byte length, not character count. Multibyte characters (e.g., emojis ðŸš€, accented chars Ã©) count as 2-4 bytes each. The counter shows actual byte usage.
  • Client-Side Only: All operations occur within the browser's JS engine. No data is transmitted, ensuring privacy, but keys are resident in browser memory during execution.
